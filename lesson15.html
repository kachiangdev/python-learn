Lesson 15: Data Structures

ğŸ—‚ï¸ What are Data Structures?

Data structures are specialized ways to organize and store data so you can access and modify it efficiently. Think of them as different types of containers, each designed for specific tasks!

ğŸ¯ Common Data Structures We'll Learn

  • Sets - Collections of unique items (no duplicates)
  • Tuples - Immutable ordered collections
  • Stacks - Last In, First Out (LIFO) collections
  • Queues - First In, First Out (FIFO) collections
  • Deques - Double-ended queues for efficient operations
  • Collections - Specialized containers for specific needs

ğŸ¯ Step 1: Sets - Unique Collections

Sets are like mathematical sets - they contain unique elements with no duplicates. Perfect for removing duplicates and checking membership!

# Creating sets fruits = {"apple", "banana", "orange", "apple"} # Duplicate "apple" will be removed print(f"Fruits set: {fruits}") # Output: {'apple', 'banana', 'orange'} # Set operations numbers = {1, 2, 3, 4, 5} more_numbers = {4, 5, 6, 7, 8} print("ğŸ”¢ SET OPERATIONS DEMO") print("=" * 40) # Union (combine sets) union_result = numbers | more_numbers # or numbers.union(more_numbers) print(f"Union: {union_result}") # Intersection (common elements) intersection_result = numbers & more_numbers # or numbers.intersection(more_numbers) print(f"Intersection: {intersection_result}") # Difference (elements in first set but not second) difference_result = numbers - more_numbers # or numbers.difference(more_numbers) print(f"Difference: {difference_result}") # Symmetric difference (elements in either set, but not both) sym_diff_result = numbers ^ more_numbers # or numbers.symmetric_difference(more_numbers) print(f"Symmetric Difference: {sym_diff_result}") # Practical example: Remove duplicates from a list original_list = [1, 2, 2, 3, 3, 3, 4, 4, 5] unique_list = list(set(original_list)) print(f"\nOriginal: {original_list}") print(f"Unique: {unique_list}") # Check membership (very fast!) if "apple" in fruits: print("ğŸ Apple is in the fruits set!") # Add and remove elements fruits.add("grape") fruits.remove("banana") # Raises error if not found fruits.discard("kiwi") # Doesn't raise error if not found print(f"Modified fruits: {fruits}")

ğŸ¯ Step 2: Tuples - Immutable Sequences

Tuples are like lists, but they can't be changed once created. Perfect for data that shouldn't be modified!

# Creating tuples coordinates = (10, 20) # 2D point rgb_color = (255, 128, 0) # Orange color person_info = ("Alice", 25, "Engineer") # Name, age, job print("ğŸ“ TUPLE EXAMPLES") print("=" * 30) # Accessing tuple elements print(f"Coordinates: x={coordinates[0]}, y={coordinates[1]}") print(f"Color RGB: Red={rgb_color[0]}, Green={rgb_color[1]}, Blue={rgb_color[2]}") # Tuple unpacking (very useful!) name, age, job = person_info print(f"Person: {name}, {age} years old, works as {job}") # Multiple assignment using tuples x, y = 10, 20 # Same as x, y = (10, 20) print(f"x={x}, y={y}") # Swapping variables (elegant Python way!) a, b = 5, 10 print(f"Before swap: a={a}, b={b}") a, b = b, a # Swap using tuple unpacking! print(f"After swap: a={a}, b={b}") # Tuples as dictionary keys (because they're immutable) locations = { (0, 0): "Origin", (10, 20): "Home", (50, 100): "Office" } print(f"Location at (10, 20): {locations[(10, 20)]}") # Named tuples (advanced feature) from collections import namedtuple # Create a named tuple class Point = namedtuple('Point', ['x', 'y']) Student = namedtuple('Student', ['name', 'age', 'grade']) # Use named tuples point = Point(10, 20) student = Student("Bob", 16, 85) print(f"\nNamed tuple point: {point}") print(f"Point x: {point.x}, Point y: {point.y}") print(f"Student: {student.name}, Age: {student.age}, Grade: {student.grade}") # Convert to dictionary student_dict = student._asdict() print(f"Student as dict: {student_dict}")

ğŸ¯ Step 3: Stacks - Last In, First Out (LIFO)

Stacks work like a pile of plates - you add to the top and take from the top. Great for undo operations, function calls, and parsing!

# Stack implementation using list class Stack: """A simple stack implementation using a list""" def __init__(self): self.items = [] def push(self, item): """Add item to the top of the stack""" self.items.append(item) print(f"ğŸ“¥ Pushed: {item}") def pop(self): """Remove and return the top item""" if not self.is_empty(): item = self.items.pop() print(f"ğŸ“¤ Popped: {item}") return item else: print("âŒ Stack is empty!") return None def peek(self): """Look at the top item without removing it""" if not self.is_empty(): return self.items[-1] return None def is_empty(self): """Check if the stack is empty""" return len(self.items) == 0 def size(self): """Get the number of items in the stack""" return len(self.items) def display(self): """Show the current stack""" print(f"Stack: {self.items} (top -> bottom)") # Stack demonstration print("ğŸ“š STACK DEMONSTRATION") print("=" * 40) stack = Stack() # Push items onto the stack stack.push("First") stack.push("Second") stack.push("Third") stack.display() # Peek at the top print(f"ğŸ‘€ Top item: {stack.peek()}") # Pop items from the stack stack.pop() stack.pop() stack.display() print(f"Stack size: {stack.size()}") # Practical example: Parentheses checker def check_parentheses(expression): """Check if parentheses are balanced using a stack""" stack = [] pairs = {')': '(', '}': '{', ']': '['} for char in expression: if char in '({[': stack.append(char) elif char in ')}]': if not stack or stack.pop() != pairs[char]: return False return len(stack) == 0 # Test parentheses checker test_expressions = [ "(hello world)", "((()))", "([{}])", "(((", "())", "{[()]}" ] print("\nğŸ” PARENTHESES CHECKER") print("=" * 30) for expr in test_expressions: result = "âœ… Balanced" if check_parentheses(expr) else "âŒ Not balanced" print(f"{expr}: {result}")

ğŸ¯ Step 4: Queues - First In, First Out (FIFO)

Queues work like a line at a store - first person in line is first to be served. Perfect for task scheduling and breadth-first search!

from collections import deque # Queue implementation using deque for efficiency class Queue: """A queue implementation using deque for O(1) operations""" def __init__(self): self.items = deque() def enqueue(self, item): """Add item to the rear of the queue""" self.items.append(item) print(f"â¡ï¸ Enqueued: {item}") def dequeue(self): """Remove and return the front item""" if not self.is_empty(): item = self.items.popleft() print(f"â¬…ï¸ Dequeued: {item}") return item else: print("âŒ Queue is empty!") return None def front(self): """Look at the front item without removing it""" if not self.is_empty(): return self.items[0] return None def is_empty(self): """Check if the queue is empty""" return len(self.items) == 0 def size(self): """Get the number of items in the queue""" return len(self.items) def display(self): """Show the current queue""" items_list = list(self.items) print(f"Queue: {items_list} (front -> rear)") # Queue demonstration print("ğŸš¶ QUEUE DEMONSTRATION") print("=" * 40) queue = Queue() # Add people to the queue queue.enqueue("Alice") queue.enqueue("Bob") queue.enqueue("Charlie") queue.display() # Serve people from the front print(f"ğŸ‘€ Next person: {queue.front()}") queue.dequeue() queue.dequeue() queue.display() # Practical example: Simple task scheduler class TaskScheduler: """A simple task scheduler using a queue""" def __init__(self): self.tasks = Queue() self.completed = [] def add_task(self, task): """Add a task to the scheduler""" self.tasks.enqueue(task) def process_next_task(self): """Process the next task in the queue""" if not self.tasks.is_empty(): task = self.tasks.dequeue() self.completed.append(task) print(f"âœ… Completed task: {task}") return task else: print("âœ¨ All tasks completed!") return None def show_status(self): """Show current status""" print(f"ğŸ“‹ Pending tasks: {self.tasks.size()}") print(f"âœ… Completed tasks: {len(self.completed)}") if not self.tasks.is_empty(): print(f"ğŸ”„ Next task: {self.tasks.front()}") # Task scheduler demonstration print("\nğŸ“‹ TASK SCHEDULER DEMO") print("=" * 30) scheduler = TaskScheduler() # Add tasks scheduler.add_task("Write code") scheduler.add_task("Test application") scheduler.add_task("Deploy to server") scheduler.add_task("Update documentation") scheduler.show_status() # Process tasks print("\nğŸ”„ Processing tasks...") while not scheduler.tasks.is_empty(): scheduler.process_next_task() scheduler.show_status()

from collections import defaultdict, Counter, OrderedDict, deque print("ğŸ§° ADVANCED COLLECTIONS DEMO") print("=" * 50) # 1. defaultdict - automatically creates missing keys print("1ï¸âƒ£ DEFAULTDICT - Auto-creating dictionaries") word_count = defaultdict(int) # Default value is 0 for missing keys text = "hello world hello python world" for word in text.split(): word_count[word] += 1 # No need to check if key exists! print(f"Word count: {dict(word_count)}") # Group students by grade student_grades = defaultdict(list) students = [("Alice", "A"), ("Bob", "B"), ("Charlie", "A"), ("David", "B"), ("Eve", "A")] for name, grade in students: student_grades[grade].append(name) print("Students by grade:") for grade, names in student_grades.items(): print(f" Grade {grade}: {names}") print("\n" + "-" * 40) # 2. Counter - counting hashable objects print("2ï¸âƒ£ COUNTER - Counting made easy") letters = Counter("hello world") print(f"Letter frequency: {letters}") # Most common elements most_common = letters.most_common(3) print(f"Most common letters: {most_common}") # Count items in a list colors = ["red", "blue", "red", "green", "blue", "red", "yellow"] color_count = Counter(colors) print(f"Color count: {color_count}") print("\n" + "-" * 40) # 3. deque - double-ended queue print("3ï¸âƒ£ DEQUE - Efficient operations at both ends") dq = deque([1, 2, 3]) print(f"Initial deque: {dq}") # Add to both ends dq.appendleft(0) # Add to left dq.append(4) # Add to right print(f"After adding to both ends: {dq}") # Remove from both ends left_item = dq.popleft() # Remove from left right_item = dq.pop() # Remove from right print(f"Removed {left_item} from left, {right_item} from right") print(f"Final deque: {dq}") # Rotate the deque dq.rotate(1) # Rotate right by 1 print(f"After rotating right: {dq}") dq.rotate(-2) # Rotate left by 2 print(f"After rotating left by 2: {dq}") print("\n" + "-" * 40) # 4. Real-world example: Browser history print("4ï¸âƒ£ BROWSER HISTORY SIMULATOR") class BrowserHistory: def __init__(self, max_history=10): self.history = deque(maxlen=max_history) # Limited size deque self.current_index = -1 def visit(self, url): """Visit a new URL""" # Remove forward history when visiting new page while len(self.history) > self.current_index + 1: self.history.pop() self.history.append(url) self.current_index = len(self.history) - 1 print(f"ğŸŒ Visited: {url}") def back(self): """Go back in history""" if self.current_index > 0: self.current_index -= 1 current_url = self.history[self.current_index] print(f"â¬…ï¸ Back to: {current_url}") return current_url else: print("âŒ No previous page") return None def forward(self): """Go forward in history""" if self.current_index < len(self.history) - 1: self.current_index += 1 current_url = self.history[self.current_index] print(f"â¡ï¸ Forward to: {current_url}") return current_url else: print("âŒ No next page") return None def show_history(self): """Show browsing history""" print("ğŸ“š Browser History:") for i, url in enumerate(self.history): marker = " <- CURRENT" if i == self.current_index else "" print(f" {i+1}. {url}{marker}") # Browser history demo browser = BrowserHistory() browser.visit("google.com") browser.visit("github.com") browser.visit("stackoverflow.com") browser.visit("python.org") browser.show_history() browser.back() browser.back() browser.visit("reddit.com") # This will clear forward history browser.show_history()

ğŸ’¡ Data Structure Comparison

ğŸ“‹ When to Use Each Structure

  • List: When you need ordered, mutable data
  • Tuple: When you need ordered, immutable data
  • Set: When you need unique items and fast membership testing
  • Dict: When you need key-value mappings
  • Stack: For LIFO operations (undo, parsing)
  • Queue: For FIFO operations (task scheduling)

âš¡ Performance Characteristics

  • List: O(1) append, O(n) insert/delete
  • Set: O(1) add/remove/lookup on average
  • Dict: O(1) get/set/delete on average
  • Deque: O(1) append/pop from both ends
  • Tuple: O(1) access, immutable
